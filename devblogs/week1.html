Week 1 - Setup & Point Generation

May 28, 2023

Logged Hours: 15

Week of 5/22: Project setup and point generation along a line.

  • Finding city assets or building a city of cubes in Unreal, finding a grapple gun asset, setting up on Perforce, marking parts of the city as grabbable or hookable.
  • Given a start and end location and a desired distance between points, generate the necessary number of points along the line. Should have a visual in game as small spheres for now, with straight lines connecting each point.
  • (Bonus) Add the basic mechanic of firing the grapple gun and, using a raycast to find an anchor point, latching on. Generate the points on this line.

This week started with basic project setup. I'm using Unreal Engine 5.1 for this project, and began with their first person C++ template. I made some adjustments to their character - adding a sprint and a horizontal speed-based FOV change, as well as setting the grapple gun to be a default component - but largely left it alone. I also made a "city" map for demoing the grapple from the level prototyping tools, and set up a simple respawn system for if the player ever swings themselves off of the map. For the grapple gun asset, I'm repurposing the shooter gun that comes with the first person template.

For point generation, I divide the distance between a start and end location by the desired distance between generated points, take the integer ceiling of that, and say that's the number of "rope segments" I have, where the number of points is just one more than the number of segments. The actual distance between points then is almost always slightly smaller than the inputted desired distance, but in a situation where the exact request is impossible, I'd rather default to giving the user more accuracy over less. Actual point placement is done just by multiplying the direction vector from start to end locations by the real distance between points and using that as an increment value for the position of points being generated.

The visuals are currently generated with debug spheres and a line batcher creating line segments between each point. The line batcher could in theory work in release builds, but it isn't well optimized for it and is just meant as a stand-in until a later week when I focus on creating the curved visuals.

I also set up the mapping of the grapple gun itself to the line generation. Physics simulation will not begin until the player has "latched on," so finding anchor points is done with a set of sphere casts. Originally, I had used only a single sphere cast, but this comes with one of two issues: either the grapple is inaccurate at close range, or it is too accurate at long range, which leads to perceived failure. At long range, the player should be able to grapple on things they think they successfully reached, which means a much larger sphere cast than would be necessary for short range. This is especially important for fast, movement-based environments. However, having a large sphere cast radius at short range produces other inaccuracies; the impact point of a sphere may be very different than what the player expects, especially at angled views.

In order to resolve this, I broke the sphere cast into several parts. The user specifies a minimum and maximum cast radius, as well as how many segments to use, and then I loop through all the inbetween radii with early outs for finding a blocking hit. This worked well; the difference is especially obvious in angled instances such as the above before and after images.

Sphere casts are also of course done centered on the player's camera view for perceived accuracy, and limited to a custum trace channel for optimization. Also, although the point generation has been bound to the player's grapple gun, the player themselves has not yet been bound to the line since no rope restraints have been put on it yet.

Next up will be putting those restraints, as well as individual velocities, on the points using Verlet Integration. By next week I hope to have a rough draft of this - the player should be held firm on one side of the rope, and the points should droop with gravity if the player moves closer.
