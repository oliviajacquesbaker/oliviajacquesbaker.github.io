Modular Card Ability System

Last worked on in May 2023

This system was developed for Get in the Box Studio's ongoing Project Phoenix, which is a deck building roguelite. In order to be as efficient in card creation as possible, we designed cards to be composite objects made of one or many modular pieces. These modular pieces hold the actual functionality of the ability associated with the card. Combined with the way most components had any number of configurations - either from component specific options or by combination with the elemental status system the game had - this allowed many more unique ability cards to be made than programmers had bandwidth for.

An example of this modularity is as follows: a card may have a ranged base component with whatever projectile type is chosen, plus a headshot component that allows the ranged projectile to score headshots, plus a status application component that makes the ranged projectile stun any enemy it hits.

On top of my contributions to the design of this system and the many card components I created, I also created the card useage pipeline. Cards are stored in decks, which exhibit all the normal cycling and shuffling functionality, etc. The player choosing to "use" a deck will use the top card of that deck, which in turn activates all the card components within that card appropriately.
