Game Engine

Last worked on in April 2023

This game engine was created as an educational exercise. Throughout it, I learned about creating iterators, using smart pointers and union members, RTTI in C++, utilizing move semantics, and much more. Throughout the project I used design patterns such as the cloneable pattern, the factory pattern, the singleton pattern, the observer pattern, and the chain of responsibility pattern.

In the end, the engine was marked complete with:

  • 3 custom containers - a templated singly linked list, a templated vector, and a templated hashmap.
  • Iterators for all custom containers - forward iterators for the list and the hashmap and a random access iterator for the vector.
  • A custom engine-specific variable used to represent game objects, all of which implement engine-specific RTTI to avoid using the global C++ system.
  • A mapping of the engine-specific variable type to key names as part of a dynamic hierarchical database.
  • Engine specific variables that represent 'actions,' such as incrementing other variables or performing a while loop.
  • An event system that uses callback functions to avoid multiple inheritence and pending lists to avoid iteration invalidation.
  • A method of automatically generating the engine-specific schema of a user-created class at compile-time using attribute lists.
  • A factory capable of creating any registered class based on its class name, for use in "blueprinting."
  • A generic parsing system, and a parsing specification to decode Json tables representing engine-specific game objects and actions, for use in "blueprinting."
  • Exhaustive unit testing to ensure correctness and completeness.

As a disclaimer, the GameTime, GameClock, and base RTTI files were provided to me; integration and tests were completed by myself.
