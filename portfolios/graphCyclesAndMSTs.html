Graphs: Finding Cycles and MSTs

Last worked on in September 2021

This project was created as algorithmic practice with graphs. The problem statement I set out to solve was as follows:

Given an undirected graph, write an algorithm to find a cycle. You can not assume the graph is connected. Given a sparse undirected graph (n nodes and at most n+8 edges), write an efficient algroithm to find the minimum spanning tree.

Cycle discovery is done using a BFS searching for a node that has already been marked as visited and then backtracking up the recorded parents of the relevant nodes to mark all nodes of the cycle.

For discovering Minimum Spanning Trees, cycles are repeatedly found within a graph, with the most expensive edge of the cycle removed each time. It should be noted that this method was choosen due to the specification of the graphs being sparse; it is a poor algorithm for dense graphs.
