Weighted Common Sub-Sequences

Last worked on in November 2021

This project was created as algorithmic practice. The problem statement I set out to solve was as follows:

Given two input strings, not necesarily of the same length, based on alphanumeric characters [A-Z], determine the best common sub-sequence. The score that determines the best sub-sequence is defined as the sum of the weights wl for each char in the sequence (i.e. wA is the weight of matching letter A) and a penalty -delta for each mismatch. Gaps are NOT allowed in the solution.

I solved this problem with dynamic programming on a bottom-up approach. At each step, the algorithm determines the maximum score between starting a new sub-sequence at the currently considered indices or adding the currently considered indices to the previous best sub-string. Since I only ever considered "this" or "the very last thing," I only had to store "the very last thing" for lookup rather than an entire possibility matrix.
